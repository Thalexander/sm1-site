Tools and Techniques


Scrum

We follow the Scrum methodology of software development!

Visual Studio Team Services

We use Visual Studio Team Services to organise our project!

Git

We use git to handle source control so we can simultaneously and remotely work on the same code without issues!

Skype

We use Skype for Business to communicate as a team!

Comunication

We used Skype for business to handle much of our team communication, as well as some meetings with our clients. Over the course of the project we also used Telegram as a more informal discussion channel, Whatsapp to communicate with other groups working on Spot Market, and Slack before we had access to Skype for business. We also used email extensively to communicate with multiple stake holders in the project. By having separate formal and informal channels of communication in the team it allowed us to more easily differentiate between discussion and plans and allowed us to much more rapidly find discussion points to refer back to due to the minimum excess messages.


File Storage

OneDrive was used as a main storage drive for all project files so we all had access to whatever files we needed and could refer back to them, however we also used Google Drive as it allowed easier collaboration on documents, so that was used to supplement OneDrive for individual sections of the project, for example making the video.


Version Control

Git was the natural version control system as we used Visual Studio Team Services to plan out our tasks and assign them, given VSTS supported it and we were all experienced in it. We also used GitHub for some parts of the development as Azure was not able to successfully able to connect to UCL's VSTS instance for automated deployment.


Integrated Development Environment

Visual Studio was used as the IDE for developing the bot as it offered integration features beyond other text editors or IDEs. For the api, standard text editors were used as there were no unique benefits of an IDE to the development.
