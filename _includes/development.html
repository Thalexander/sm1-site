Testing Methods

In order to see if we can provide a great user experience for consumers, we will use multiple different testing methods. As we are planning to use the Microsoft bot framework using Node.js, we will need to look into the testing strategies available for this platform.

Unit Testing

Unit testing allows the developer to check small “units” of the project using unit test functions. For Node.js in particular, one of the famous unit test frameworks is Mocha. Mocha is open source, simple to use and is very easy to learn. We are planning to use Mocha for testing the functional units of the bot during the development process.

Integration Testing

Integration testing tests the communication between units of the code, in order to show that all units work while interacting with each other. Mocha and Chai also support integration testing of a project, and we are hoping to be able to use them during our development process.

Acceptance Testing

As we develop the chatbot, we intend to perform user acceptance testing to monitor whether we are meeting the requirements that the client has set out. This will involve putting the bot out to regular users as well as the client to ensure that the bot behaves according to specifications and that users can easily interact with the bot.
